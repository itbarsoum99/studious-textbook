Webtextbook

These are my notes for the textbook, Learning Web Design by Jennifer Robbins.

Heading

Headings used to create a document outline

Headings range from <h1> to <h6>.

Paragraph

Paragraphs are used to present body text

Paragraphs are indicated with the tag <p>

Unordered List

List with items in no particular order

<ul> = unordered list
<li> = list item
  • Serif
  • Sans-serif
  • Script

Ordered List

List where items' order is important

<ol> = ordered list
<li> = list item
  1. Math
  2. Spanish
  3. History

Description List

List where items are name/value pairs

<dl> = description list
<dt> = a name, such as a term or label
<dd> = a value, such as a description or definition
Math
The branch of science dedicated to values and figures.
Spanish
The second most common language worldwide, it is the official language in 21 different countries from Spain to Equatorial Guinea.
History
The study of past events and how they relate to current, global ideas.

Blockquote

Used for long quotations, testimonials, or a copied section from another source

<blockquote>

Our alphabet hasn't changed in eons; there isn't much latitude in what a designer can do with the individual letters.

Much like a piece of classical music, the score is written down,It's not something that is tampered with, and yet, each conductor interprets that score differently. There is tension in the interpretation.

Preformatted Text

Used to make whitespace semantically significant

<pre> 
    This is           an          example of
        text with a      lot of
                         curious
                         whitespace.

This is an example of text with a lot of curious whitespace.

(example shows <pre> vs <p> elements)

Figure

Used to identify content semantically

<figure> 
    <figure>
      <img src="https://external-content.duckduckgo.com/iu/?u=https%3A%2F%2Fi.ytimg.com%2Fvi%2FkJ2dr9jAThY%2Fmaxresdefault.jpg&f=1&nofb=1" alt="hello">
    </figure>
<img> = image (within figure element) <figcaption> = caption for a figure <code> = code rule
